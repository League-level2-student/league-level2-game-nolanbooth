Territorial.io - Kind Of

by: Nolan Booth

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I made this game because I enjoy playing territorial.io. In the beginning I had no idea what game to make, and it was around then when I started to get addicted to territorial, so I decided to make it.

2. What I Learned

I learned that making this game wasn't as simple as I thought, and now that I know how it works, I respect the developers of the actual game a whole lot more now. It seems like a really simple coding process when you look at it, with its simple graphics and straightforward gameplay. While this might be true in comparison to games like Minecraft or something, there is still a large amount of skill involved in making it.

3. Architecture of My Game

There are different classes controlling the nations, tiles, and general game stuff. For example, when you attack something, the input goes from the GamePanel class to the Nation class, where the attack is computed. Then tiles are taken using code from the Tile Class. The Nation class handles attacks as well as other things having to do with the nations, such as computing troops and their interest. The GamePanel class is where all the user input and timers are, as well as the methods for drawing the game. The WorldManager class manages the world, and it is where all the bots, players, tiles, and world gets created. The Runner class runs the program. The attack class stores an attack variable.

4. Technical Implementation

I am very proud of the system that detects the neighbors of a certain nation, because it took a long time to make. In real life, it is very easy to tell whether two things are touching each other, but a computer doesn't have eyes so it can't see what is touching what. So, we have to use different methods of telling the computer which tile is touching another tile. This was very messy and complicated, but worked out in the end.

5. Most Challenging Part

The most challenging part of the project was getting the Attack method to work properly, as well as computing the neighbors of the tiles. Overall, I would say the attack method was more difficult, since it also involved a little bit of the tile stuff. Making the attacks continue when there was still troops left took a long time to get right.

6. Future Fixes and Updates

If I were to continue working on this game, I would improve the AI of the bots, as well as the balancing of the game. I had to give the player a lot of edge of the bots, since they would all team up on you and destroy your nation pretty easily.
